# Gemini 위임 (gemini-delegate)

너(오케스트레이터)는 작업 전체를 책임진다. 위임은 하위 작업 하나를 워커(Gemini CLI)에
넘기고 결과만 회수하는 것이며, task 전체를 넘기는 라우팅이 아니다.

## 1. 위임 적합성 판정 — 속성 테스트

하위 작업이 **네 가지 속성을 모두** 만족할 때만 위임하라. 난이도가 아니라 속성으로 판정한다.

1. **Spec 완결성** — 대화 맥락 없이 자체 완결된 지시문을 한 번에 쓸 수 있다.
   ("적당히", "지금까지 논의한 대로" 같은 암묵 맥락이 필요하면 부적합)
2. **검증 비용 ≪ 수행 비용** — 결과가 맞는지 확인하는 비용이 직접 하는 비용보다 훨씬 싸다.
3. **재시도 저렴** — 실패해도 다시 시키거나 회수하는 비용이 낮다 (부분 실패가 다른 작업을 오염시키지 않는다).
4. **보안 무관** — 자격증명, 비밀키, 민감 데이터를 다루지 않는다.

### 위임 금지 (네거티브 리스트)

- 설계 판단·아키텍처 결정·트레이드오프 분석이 필요한 작업
- spec이 모호하거나 "알아서" 판단해야 하는 작업
- 결과 검증에 결국 전체를 다시 봐야 하는 작업 (검증 비용 ≈ 수행 비용)
- 실패 시 재작업 비용이 큰 작업 (핵심 로직 수정, 마이그레이션 등)
- 자격증명·비밀·개인정보를 읽거나 쓰는 작업

### 카탈로그 (예시)

| 적합 | 부적합 |
|---|---|
| 수십 개 파일 탐색·구조 요약 | 모듈 인터페이스 설계 |
| 대량 로그/빌드 출력에서 오류 패턴 추출 | 버그 원인 판정 (증거 수집까지만 적합) |
| 정형 보일러플레이트 생성 (테스트 골격, DTO 등) | 핵심 비즈니스 로직 구현 |
| 기계적 반복 수정 (rename, import 정리, 포맷 통일) | 리팩토링 방향 결정 |
| 문서 초안·번역·표 변환 | 커밋 메시지 (맥락 필요, 직접 쓰는 게 더 쌈) |

## 2. Spec 작성 템플릿

위임 spec은 자체 완결이어야 한다. 다음 구조로 작성하라:

```
[목표] 무엇을 왜 하는지 한 문단
[대상] 구체적 경로/파일 패턴 (워커가 직접 읽는다 — 내용을 붙여넣지 마라)
[출력] 기대하는 출력 형식과 분량
[제약] 하지 말 것, 경계 조건
```

근거 인용·수정 범위 제한 같은 위임 계약은 delegate.py가 자동 첨부하므로 spec에 반복하지 마라.

## 3. 실행

이 스킬 디렉토리의 delegate.py를 사용하라 (`<skill-dir>`은 이 SKILL.md가 있는 디렉토리):

```bash
# 읽기 위임 (탐색·요약·분석) — 읽기 전용이 플래그로 강제됨
python "<skill-dir>/delegate.py" --type read "spec..."

# 기계적 쓰기 위임 — 파일 수정만 자동 승인됨 (임의 셸 실행 불가)
python "<skill-dir>/delegate.py" --type write --spec-file spec.md

# spec이 길면 --spec-file 또는 stdin을 써라 (인용 문제 회피)
```

- 결과는 stdout으로 반환된다. 위임은 blocking이다.
- 기본 타임아웃 600초, `--timeout`으로 조정. 모델은 `--model`로 오버라이드.
- 모든 위임은 `~/.claude/gemini-delegate/ledger.jsonl`에 자동 기록된다.

## 4. 결과 검증 — 생략 불가

- **읽기 위임**: 이후 작업의 전제가 되는 하중 큰 주장을 골라, 인용된 `경로:줄번호`를
  직접 읽어 스팟체크하라. 근거 인용이 없는 주장은 검증 불가로 취급하고 사용하지 마라.
- **쓰기 위임**: `git diff`를 반드시 검토하라 (git 저장소가 아니면 워커가 보고한
  수정 파일을 직접 읽어 확인하라). 검토 전에는 완료로 취급하지 마라.

## 5. 실패 처리 — 출구 규칙

- **검증 탈락 1회차**: 무엇이 틀렸는지 피드백을 spec에 담아 `--retry`로 정확히 1회 재위임.
- **재탈락**: 회수하라 — 네가 직접 수행한다. 추가 재시도 금지.
- **워커 불능** (쿼터 소진·타임아웃·실행 오류): 재시도 없이 즉시 회수하고,
  사용자에게 한 줄로 고지하라 (예: "Gemini 쿼터 소진으로 직접 수행합니다").
