# 데이터 경계는 배포 전제가 담보하고, 위임 차단은 시크릿만 본다

사내망 오케스트레이터가 속성 4("보안 무관 — 자격증명, 비밀키, 민감 데이터")의 '민감 데이터'를
사내 코드 전반으로 확대 해석해, 코드 심층 분석 위임을 "외부 Gemini로의 내부 데이터 유출"로
과잉 차단하는 사고가 있었다. 이 하네스의 배포 환경은 사내망 전용(ADR 0002)이고
오케스트레이터·워커(Claude Code/Codex/Gemini CLI) 모두 사내 enterprise 계약 아래에 있으므로,
일반 사내 코드·문서·로그를 워커에 전달하는 것은 데이터 유출이 아니다. 따라서 속성 4를
**시크릿 무관**으로 재정의한다: 판정 대상은 spec 텍스트와 대상 경로에 시크릿(자격증명·비밀키·
토큰류)이 포함되는지 여부뿐이며, 대상 트리에 시크릿 파일이 섞여 있으면 통짜 부적합이 아니라
spec [제약]의 경로 제외로 구제한다. 강제는 SKILL.md 판정 규칙(문구)으로만 하고 delegate.py에
기계 강제는 넣지 않는다.

## Considered Options

- **광역 차단 유지('민감 데이터' 존치)** — 열린 용어가 "사내 데이터 전부"로 읽혀 사내망에서
  사실상 모든 위임이 막힌다. 하네스의 존재 이유(고성능 토큰 절약)를 무력화하므로 기각.
- **개인정보를 차단 목록에 유지** — enterprise 계약이 커버하는 데이터 부류로 판단해 **의도적으로
  제외**했다. 차단 목록에서 개인정보가 빠진 것은 누락이 아니라 결정이다.
- **delegate.py 기계 강제(spec 시크릿 패턴 스캔·대상 트리 사전 스캔)** — 시크릿 판별은 맥락
  의존적이라 정규식 오탐 시 위임 자체가 막히고, spec에 파일 내용을 붙여넣지 않는 구조라 spec
  스캔의 실익이 적다. 트리 사전 스캔은 위임마다 순회 비용이 들어 하네스 목적(가벼움)과 충돌. 기각.
