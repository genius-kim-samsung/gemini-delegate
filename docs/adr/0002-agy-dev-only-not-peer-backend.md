# agy는 peer 백엔드가 아니라 사외망 개발 전용 탈출구

이 스킬은 사내망 전용으로 운영되며 사내망에는 Gemini CLI만 존재하고 agy(Antigravity CLI)는
없다. 그러나 유지보수는 gemini에 접근할 수 없는 사외망(집)에서 하므로, 그곳의 테스트를 위해
agy가 필요하다. 그래서 정식 경로는 **gemini 전용**(자동 감지 없음, 다른 백엔드로의 폴백 없음)으로
두고, agy는 환경변수 `GEMINI_DELEGATE_DEV_AGY=1`이 켜진 개발 머신에서만 열리는 탈출구로 강등했다.
스위치가 꺼진 상태에서 `--backend agy`는 하드 에러로 거부되어 정식 코드 경로에 agy가 새어
들어가지 않는다. 이는 agy를 gemini와 동등한 백엔드로 올렸던 이전 설계(커밋 `689e69d`,
`9927d4e`: auto 감지 기본화·폴백 지시)를 의도적으로 뒤집는 결정이다.

## Considered Options

- **peer 백엔드 + auto 감지 + gemini 실패 시 agy 폴백 (기존)** — 정식 정책("gemini 전용")을
  흐리고, 사내망엔 agy가 없어 폴백 지시가 항상 헛돌며 오케스트레이터에게 잘못된 안내가 됨. 기각(반전).
- **명령마다 디버그 플래그(`--dev` 등)** — 사외망에선 모든 위임이 agy로 가야 해 매 호출에 반복 부담이
  생기고, SKILL.md가 그 플래그를 가르쳐야 해서 정식 지시문에 디버그 경로가 누출됨. 기각.
- **환경변수 dev 스위치 (채택)** — 개발 머신에 한 번만 설정. SKILL.md의 명령은 두 환경에서 동일하고
  라우팅은 delegate.py가 스위치로 투명 처리하므로, 오케스트레이터는 agy의 존재를 몰라도 된다.
