# 탐색 위임 래퍼는 Claude Code 전용으로, setup 스킬이 배포한다

탐색 위임을 오케스트레이터의 선의에 맡기지 않고 강제하려면 "직접 탐색 금지"를 규칙으로 못박은
서브에이전트(`Explore`)가 필요하다. 그런데 이건 **Claude Code에만 있는 개념**이고
(Codex CLI 0.144에는 정의 파일 기반 서브에이전트가 없다 — plugin·mcp만 있다), 이 저장소가
지켜 온 호스트 대칭을 깬다. 그럼에도 배포하기로 하고, 비대칭을 숨기지 않는다 —
`setup-gemini-delegate`가 호스트를 감지해 Claude Code면 `~/.claude/agents/Explore.md`를
전역에 쓰고, Codex면 건너뛰며 그 사실을 한 줄 고지한다. 자동 트리거와 같은 운반체를 쓰는 셈이라
설치 단계가 늘지 않고, `docs/adr/0004`가 세운 "메모리 파일 같은 비-스킬 자리는 setup 스킬이
소유한다"는 경계도 그대로 유지된다.

## Considered Options

- **플러그인으로 배포** — 서브에이전트 배포의 유일한 공식 채널이고 가장 깔끔하다. 그러나 사내망에서
  보안 정책상 플러그인 설치가 차단되어 있어 **주 사용 환경에서 못 쓴다**. 기각.
- **`npx skills`로 배포** — 현행 채널이지만 스킬 디렉터리만 복사한다. CLI에 `--subagent`
  플래그가 있으나 Eve 호스트의 subagent 폴더에 *스킬*을 넣는 용도라 무관하다. 구조적으로 불가. 기각.
- **스킬 디렉터리의 템플릿으로 심링크** — `npx skills update`가 자동 전파된다는 이점이 크다.
  그러나 (a) 파일을 못 고치므로 `delegate.py` 절대경로 치환이 불가능해지고, (b) Claude Code가
  `agents/` 안의 심링크를 인식하는지 **문서에 명시가 없으며**, (c) Windows에서 심링크 생성이
  권한을 탄다. 미확인 리스크 셋을 안느니 복사+치환을 택했다. 기각.
- **README에 수동 복사 안내만** — 구현 비용 0이지만 경로 치환·갱신·제거를 전부 사용자가
  손으로 떠안는다. 기각.
- **setup 스킬이 템플릿을 읽어 치환 후 복사 (채택)** — 경로가 OS·설치위치 무관하게 정확해지고,
  갱신·제거가 마커로 멱등하게 돌아간다. 대가로 `npx skills update` 후 `/setup-gemini-delegate`
  재실행이 필요하며, 이를 README에 명시한다.

## Consequences

- **컨텍스트 격리는 얻지 못한다.** 래퍼는 워커 stdout을 원문 그대로 되돌리므로 오케스트레이터가
  받는 분량은 직접 위임했을 때와 같다. 요약을 시키면 근거 인용(`경로:줄번호`)이 유실되어
  스팟체크 계약이 무너지므로 일부러 하지 않는다. 실제 이득은 직접 탐색 금지의 강제와,
  spec 작성·bash 왕복·stderr 처리를 저비용 모델(haiku)이 대신하는 출력 토큰 절약뿐이다.
  문서가 "격리"를 약속하지 않게 한다.
- **이름 `Explore`를 전역에서 선점한다.** 흔한 이름이라 사용자가 이미 갖고 있을 수 있다.
  우리 파일에만 `<!-- gemini-delegate:managed -->` 마커를 넣고, 마커 없는 동명 파일은
  덮어쓰지도 지우지도 않는다 — 그 사용자는 직접 결정하지 않는 한 이 기능을 못 받는다. 수용한다.
- **호출 유도 문장은 넣지 않는다.** 서브에이전트 description만으로 발동이 관찰되어, 자동 트리거
  블록을 호스트별로 분기시키는 비용을 피했다. 실사용에서 미발동이 쌓이면 `gemini-delegate`
  SKILL.md 실행 절에 한 줄 넣는 것이 다음 후보다(메모리 파일이 아니라).
