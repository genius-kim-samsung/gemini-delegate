# CLI 위임 하네스 (CLI Delegation Harness)

고성능 CLI(Claude Code/Codex)가 메인 에이전트로 작업하면서, 저성능이지만 토큰이 풍부한 Gemini CLI에 하위 작업을 위임해 월별 토큰 한도를 절약하는 하네스.

## Language

**오케스트레이터 (Orchestrator)**:
작업 전체를 책임지고 수행하며, 하위 작업의 위임 여부를 판단하는 고성능 에이전트(Claude Code 또는 Codex CLI).
_Avoid_: 라우터, 분류기, 디스패처

**워커 (Worker)**:
오케스트레이터로부터 하위 작업을 위임받아 실행하는 저비용 에이전트(Gemini CLI).
_Avoid_: 서브에이전트(구현 형태를 지칭할 때만), 저성능 모델

**위임 (Delegation)**:
오케스트레이터가 하위 작업 하나를 워커에게 넘기고 결과만 회수하는 행위. task 전체를 넘기는 라우팅과 구별된다.
_Avoid_: 라우팅, 오프로딩

**기계적 쓰기 (Mechanical Write)**:
판단이 아니라 패턴 적용만 필요한 파일 수정(보일러플레이트 생성, 정형 반복 수정 등). 워커에게 위임 가능한 쓰기 작업의 상한이며, 오케스트레이터의 diff 검토를 통과해야 완료로 인정된다.
_Avoid_: 단순 수정, 잡일

**위임 적합성 (Delegability)**:
하위 작업을 워커에게 맡겨도 되는 정도. task 난이도가 아니라 spec 완결성(자체 완결된 지시문 작성 가능 여부)과 검증 비용(결과 확인 비용 ≪ 직접 수행 비용)으로 판정한다.
_Avoid_: 난이도, 단순함

**근거 인용 (Evidence Citation)**:
워커가 읽기 위임 결과의 모든 주장에 첨부해야 하는 file:line 출처. 이것이 없는 주장은 검증 불가로 취급한다.

**스팟체크 (Spot-check)**:
오케스트레이터가 워커 결과 중 이후 작업의 전제가 되는 하중 큰 주장만 골라 인용 위치를 직접 읽어 확인하는 검증 행위.
_Avoid_: 전수 검사, 리뷰(diff 검토와 혼동 주의)

**회수 (Reclaim)**:
위임이 실패(검증 탈락 1회 재시도 후 재탈락, 또는 워커 불능)했을 때 오케스트레이터가 해당 하위 작업을 직접 수행하는 것으로 전환하는 행위.
_Avoid_: 폴백, 롤백

**위임 장부 (Delegation Ledger)**:
delegate 스크립트가 매 위임마다 남기는 JSONL 기록(시각, 유형, spec 요약, 결과, 소요 시간). 위임 기준 튜닝과 하네스 효과 판단의 근거.
_Avoid_: 로그(일반 디버그 로그와 혼동 주의)
